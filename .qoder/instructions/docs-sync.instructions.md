# Documentation Sync

- 本文件用于补充文档同步细则，不替代 AGENTS.md 中的全局文档同步义务。
- 新增、修改、删除代码、配置、依赖、接口、入口、示例或目录结构时，必须把"同步更新文档"视为同一次变更的一部分，不能留到后续补做。
- 若本次改动属于缺陷修复、兼容性问题修补、行为偏差修正、边界条件修正或回退逻辑补丁，必须同步检查相关文档是否需要补充问题背景、修复影响、使用限制、行为变化或排查方式。
- 任何会影响理解方式、使用方式、配置方式、运行入口、依赖关系、扩展方式或目录职责的改动，都必须检查受影响文档。
- 若目标项目目录、模块目录、功能目录或示例目录已存在 README、设计文档、迁移文档或说明文件，优先更新现有文档，不得新增平行说明文件。
- 若上述目录没有说明文档，且本次改动已达到需要外部维护者理解或接手的程度，应补充最小 README。
- 最小 README 至少覆盖：目录用途、核心入口、依赖关系或前置条件、运行或调试方式、本次改动影响点、维护注意事项。
- 文档补充只在项目目录、模块目录、功能目录、示例目录层级考虑；第三方依赖、构建输出、自动生成、缓存和二进制目录不在范围内。
- 对纯格式化或不影响外部理解的内部微调，可不新增文档；但若现有文档、注释、示例、命令、类名、路径、配置项、接口行为或目录说明已与实现不一致，必须同步修正。
- 若文档中仍保留旧行为描述、旧限制、旧示例、旧修复说明或旧排查方式，必须在本次修复中一并修正。
- 文档更新时必须核对 README、说明文档、示例代码、命令、接口说明、配置说明、路径引用和架构描述，确保它们与当前代码一致。
- 提交前必须进行一次代码与文档一致性检查，确认示例、命令、类名、文件路径和配置项与当前实现一致。
