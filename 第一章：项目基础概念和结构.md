# 第一章：项目基础概念和结构

## 1.1 什么是 Android 项目

Android 项目是用来开发能在 Android 手机上运行的应用程序的工程文件集合。就像你要做一道菜需要食材和菜谱一样，要做一个手机 App，你需要代码文件和配置文件。

你可以把 Android 项目想象成一个房子的建筑图纸和材料清单：
- 代码文件就像是建筑材料（砖块、水泥等）
- 配置文件就像是建筑图纸（告诉工人怎么建造）

## 1.2 项目目录结构

当你打开这个 AndroidProject 项目时，你会看到很多文件夹和文件。我们来逐一解释它们：

```
AndroidProject/
├── app/                 # 这是存放我们 App 主要代码的地方
├── library/             # 这里存放的是别人写好的代码库，我们可以直接使用
├── README.md           # 项目的说明书，告诉我们这个项目是干什么的
├── HelpDoc.md          # 常见问题解答，遇到问题可以在这里找答案
```

### app 文件夹详解

```
app/
├── src/main/           # 我们主要工作的目录
│   ├── java/           # 存放 Java 代码文件
│   ├── res/            # 存放资源文件（图片、布局等）
│   └── AndroidManifest.xml  # App 的身份证，记录 App 的基本信息
```

### library 文件夹详解

```
library/
├── base/               # 基础代码库
├── umeng/              # 友盟相关功能（统计、推送等）
├── widget/             # 控件库（各种好看的按钮、输入框等）
└── xbus/               # 事件总线（组件之间通信的工具）
```

## 1.3 核心组件介绍

在 Android 中，有几个核心概念你需要了解：

### Activity（活动）
可以把 Activity 想象成一个手机屏幕页面。比如：
- 微信的聊天界面是一个 Activity
- 微信的朋友圈界面是另一个 Activity
- 设置界面也是一个 Activity

在本项目中，所有的 Activity 都继承自 [AppActivity](file:///D:/git/app/AndroidProject/app/src/main/java/run/yigou/gxzy/app/AppActivity.java#L25-L230)，这样可以共享一些通用功能。

### Fragment（片段）
Fragment 是 Activity 的一部分。比如：
- 在一个新闻页面中，顶部的标题栏是一个 Fragment
- 下面的新闻列表是另一个 Fragment

在本项目中，所有的 Fragment 都继承自 [AppFragment](file:///D:/git/app/AndroidProject/app/src/main/java/run/yigou/gxzy/app/AppFragment.java#L16-L76)。

### Application（应用）
Application 代表整个应用程序，从你打开 App 到关闭 App，它一直存在。在整个 App 运行过程中，它负责初始化各种功能。

在本项目中，[AppApplication](file:///D:/git/app/AndroidProject/app/src/main/java/run/yigou/gxzy/app/AppApplication.java#L58-L317) 类就是我们的 Application，它负责：
- 初始化各种第三方库
- 管理用户的登录状态
- 设置全局配置
